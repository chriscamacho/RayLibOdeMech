Detailed Description

Initialization and cleanup functions for graphics and physics.

This file provides functions to initialize and clean up the graphics and physics subsystems. It handles resource loading, memory allocation, and proper shutdown of all framework components.

Author
Chris Camacho (codifies - http://bedroomcoders.co.uk/)
Date
2026

Initialization Order

Typical initialization sequence:

  1. InitGraphics() - Set up window, models, textures, shaders
  2. InitPhysics() - Create ODE world, space, contact groups

Cleanup Order

Typical cleanup sequence:

  1. CleanupPhysics() - Destroy ODE world, free bodies
  2. CleanupGraphics() - Unload textures, shaders, close window

Initialize graphics context and window.

Sets up the RayLib window, loads all models and textures, compiles shaders, and creates lighting. This must be called before any rendering can occur.

Parameters

PhysicsContext * InitPhysics ( )

Initialize physics world and context.

Creates and configures the ODE physics world, collision space, and contact group. Also seeds the random number generator.

Returns
Pointer to newly created PhysicsContext
Note
Checks for single-precision ODE build (required)
Sets gravity to -9.8 m/s^2 (Earth gravity)
Enables auto-disable for resting bodies
Warning
Exits with error message if precision mismatch detected
